# День 2: Сувенирная лавка

Вы попадаете внутрь и поднимаетесь на лифте на единственный другой этаж: сувенирную лавку.  
«Спасибо за посещение Северного Полюса!» — радостно гласит ближайшая табличка.  
Вы не совсем уверены, кто вообще имеет право посещать Северный Полюс, но знаете, что сможете пройти в вестибюль отсюда, а оттуда — в остальную часть базы.

Пока вы идёте через удивительно обширный ассортимент, один из продавцов узнаёт вас и просит о помощи.

Выясняется, что один из юных эльфов играл на компьютере сувенирной лавки и умудрился добавить в базу *кучу неверных идентификаторов товаров*! Конечно, вам не составит труда найти неверные ID, верно?

Они уже проверили большинство диапазонов ID; осталось всего несколько диапазонов (ваш входной файл), которые нужно проверить. Например:

```
11-22,95-115,998-1012,1188511880-1188511890,222220-222224,
1698522-1698528,446443-446449,38593856-38593862,565653-565659,
824824821-824824827,2121212118-2121212124
```

(Диапазоны перенесены для удобства чтения; в вашем вводе они находятся в одной длинной строке.)

Диапазоны разделены запятыми `,`; каждый диапазон задаётся первой и последней ID, разделёнными дефисом `-`.

Так как юный эльф просто делал глупые повторяющиеся последовательности, вы можете определить неверные ID, найдя любые числа, которые состоят из некоторой последовательности цифр, повторённой дважды. Например:

- `55` (5 дважды),
- `6464` (64 дважды),
- `123123` (123 дважды)

— все являются неверными ID.

Ни у одного числа нет ведущих нулей; `0101` не является ID вообще. (`101` — допустимый ID, и вы его игнорируете.)

Ваша задача — найти **все неверные ID**, которые встречаются в данных диапазонах.

В приведённом примере:

- `11–22` содержит два неверных ID: `11` и `22`.
- `95–115` содержит один неверный ID: `99`.
- `998–1012` содержит один неверный ID: `1010`.
- `1188511880–1188511890` содержит один неверный ID: `1188511885`.
- `222220–222224` содержит один неверный ID: `222222`.
- `1698522–1698528` не содержит неверных ID.
- `446443–446449` содержит один неверный ID: `446446`.
- `38593856–38593862` содержит один неверный ID: `38593859`.
- Остальные диапазоны не содержат неверных ID.

Сложив все неверные ID в данном примере, получается **1227775554**.

## Что вы получите, если сложите все неверные ID?

# Часть Вторая

Продавец быстро обнаруживает, что в диапазонах из вашего списка всё ещё есть неверные ID.  
Возможно, юный эльф использовал и другие забавные шаблоны?

Теперь ID считается неверным, если оно состоит из некоторой последовательности цифр, повторённой **как минимум дважды**.  
Например:

- `12341234` (1234 повторено дважды)
- `123123123` (123 повторено трижды)
- `1212121212` (12 повторено пять раз)
- `1111111` (1 повторено семь раз)

— все это неверные ID.

---

Используя тот же пример, что и ранее:

- `11–22` всё ещё содержит два неверных ID: `11` и `22`.
- `95–115` теперь содержит два неверных ID: `99` и `111`.
- `998–1012` теперь содержит два неверных ID: `999` и `1010`.
- `1188511880–1188511890` всё ещё содержит один неверный ID: `1188511885`.
- `222220–222224` всё ещё содержит один неверный ID: `222222`.
- `1698522–1698528` всё ещё не содержит неверных ID.
- `446443–446449` всё ещё содержит один неверный ID: `446446`.
- `38593856–38593862` всё ещё содержит один неверный ID: `38593859`.
- `565653–565659` теперь содержит один неверный ID: `565656`.
- `824824821–824824827` теперь содержит один неверный ID: `824824824`.
- `2121212118–2121212124` теперь содержит один неверный ID: `2121212121`.

Сумма всех неверных ID в этом примере равна **4174379265**.
